## 1. Что реализовано

### 1.1. Маршруты (`app/student.py`)

| Маршрут | Метод | Защита | Описание |
|---|---|---|---|
| `/lessons` | GET | `@login_required_student` | Список доступных уроков: только `is_open=True` и не истёкших по сроку. Сортировка по убыванию порядкового номера (новые сверху). Каждый урок — ссылка |
| `/lessons/<id>` | GET | `@login_required_student` | Страница урока с трёхфреймовой структурой |
| `/lessons/<lesson_id>/tasks/<task_id>` | GET | `@login_required_student` | AJAX-эндпоинт: возвращает HTML условия задачи для динамической загрузки в Фрейм 2 |

**Защита доступа:**
- Закрытый урок → редирект на `/lessons`
- Истёкший урок → редирект на `/lessons`
- Несуществующий урок/задача → 404
- Задача не принадлежит уроку → 404

### 1.2. Трёхфреймовая структура (раздел 6 ТЗ)

Страница урока реализована через CSS Flexbox:

```
┌────────────┬───────────────────────────────┐
│            │  Фрейм 2: Условие задачи       │
│  Фрейм 1:  │  (6 разделов + примеры)        │
│  Список    ├───────────────────────────────┤
│  задач     │  Фрейм 3: Редактор кода +      │
│  (A, B, …) │  область результатов            │
└────────────┴───────────────────────────────┘
```

| Фрейм | Назначение | Реализация |
|---|---|---|
| Фрейм 1 (левая колонка) | Список задач урока | `<ul class="task-list">`, нумерация латинскими буквами (`A. <название>`), активная задача подсвечивается. При клике — AJAX-запрос, обновление Фрейма 2 и 3 |
| Фрейм 2 (верх основной области) | Условие задачи — 6 разделов | `student/task_statement.html` (см. 1.3) |
| Фрейм 3 (низ основной области) | Редактор кода + результаты | `student/editor.html` (см. 1.4) |

### 1.3. Фрейм 2 — условие задачи (раздел 6.3)

Шаблон `student/task_statement.html` собирает условие из **6 разделов**:

| № | Раздел | Источник данных |
|---|---|---|
| 1 | Ограничения | `task.time_limit`, `task.memory_limit` (таблица: время / память) |
| 2 | Условие | `task.problem_text` |
| 3 | Входные данные | `task.input_description` |
| 4 | Результат | `task.output_description` |
| 5 | Примеры | Автогенерация из тестов 1 и 2 через `get_examples(task_id)` — таблица с двумя колонками (входные данные / результат) |
| 6 | Пояснения | `task.notes` (только если заполнено) |

AJAX-запрос `GET /lessons/<lesson_id>/tasks/<task_id>` возвращает этот фрагмент HTML для динамической подгрузки при переключении задач.

### 1.4. Фрейм 3 — редактор кода (раздел 6.4)

Шаблон `student/editor.html`:

- **Редактор кода с подсветкой синтаксиса** — **CodeMirror 5.65.16** (JS-библиотека, файлы в `static/vendor/codemirror/`)
  - Нумерация строк
  - Тема `dracula`
  - Автоподстановка закрывающих скобок (`autoCloseBrackets`)
  - Подсветка парных скобок (`matchBrackets`)
  - Отступ 4 пробела
- **Переключатель языка** — `<select>` с двумя опциями: Python и C++ (GNU GCC Compiler). При смене языка меняется режим подсветки CodeMirror (`changeLanguage()`)
- **Кнопка «Проверить»** — отправка кода через AJAX POST на `/submit`
- **Область вывода результатов** — таблица с результатами по каждому тесту, текст ошибок компиляции, итоговый статус

### 1.5. CodeMirror — подключение

Файлы библиотеки скачаны в `static/vendor/codemirror/`:

```
vendor/codemirror/
├── lib/
│   ├── codemirror.js        (170 КБ)
│   └── codemirror.css       (6 КБ)
├── mode/
│   ├── python/python.js     (подсветка Python)
│   └── clike/clike.js       (подсветка C/C++)
├── addon/edit/
│   ├── matchbrackets.js     (подсветка парных скобок)
│   └── closebrackets.js     (автозакрытие скобок)
└── theme/
    └── dracula.css          (тёмная тема)
```

Подключение в `lesson.html`:
- CSS — в блоке `{% block head %}` (в `<head>`)
- JS — в блоке `{% block scripts %}` (перед пользовательским скриптом)

### 1.6. JavaScript-логика (`lesson.html`)

| Функция | Назначение |
|---|---|
| `initEditor()` | Инициализация CodeMirror из `<textarea>`, вызывается при `DOMContentLoaded` |
| `changeLanguage()` | Смена режима подсветки при переключении языка (`python` / `text/x-c++src`) |
| `loadSavedCode()` | Загрузка сохранённого кода и языка из `Submission` (из скрытых полей, передаваемых в `task_statement.html`) |
| `loadTask(taskId)` | AJAX-запрос условия задачи, обновление Фрейма 2, активация пункта в Фрейме 1, загрузка сохранённого кода |
| `submitCode()` | Отправка кода на `/submit`, рендеринг результатов (таблица тестов, ошибки компиляции, итоговый статус) |

### 1.7. Загрузка последней попытки

При открытии задачи, если у ученика есть сохранённая `Submission`:
- Код загружается в редактор CodeMirror
- Язык устанавливается в переключателе
- Режим подсветки обновляется соответственно

### 1.8. CSS (`static/css/style.css`)

Добавлены стили для трёхфреймового layout и CodeMirror:
- `.lesson-layout` — Flexbox-контейнер (Фрейм 1 слева, Фреймы 2+3 справа)
- `.frame-tasks` — фиксированная ширина 220px, прокрутка
- `.frame-main` — колонка из условия (сверху, `flex: 1`) и редактора (снизу)
- `.CodeMirror` — заполняет доступное пространство, моноширинный шрифт

Цветовая схема: белый фон, синий акцент (`#2c5aa0`), контрастный шрифт — раздел 9.1 ТЗ.

## 4. Технические решения

- **CodeMirror вместо plain textarea** — добавлена подсветка синтаксиса для Python и C++ согласно разделу 6.4 ТЗ. Файлы библиотеки хранятся локально в `static/vendor/` (не CDN) — для работы в локальной сети без доступа к Интернету.
- **`CodeMirror.fromTextArea()`** — редактор создаётся поверх `<textarea>`, что сохраняет совместимость с тестами (проверяют наличие `<textarea>`) и обеспечивает fallback.
- **`changeLanguage()` исправлена** — ранее функция вызывалась в `onchange` селектора языка, но не была определена (JS-ошибка). Теперь она переключает режим подсветки CodeMirror (`python` ↔ `text/x-c++src`).
- **AJAX-загрузка задач** — при клике на задачу в Фрейме 1 отправляется `fetch()`-запрос, ответ (HTML условия) вставляется в `innerHTML` Фрейма 2. Редактор (Фрейм 3) при этом не перезагружается — CodeMirror сохраняет состояние.
- **Сохранённый код через скрытые поля** — `task_statement.html` передаёт `saved-code` и `saved-language` как `<input type="hidden">`. При AJAX-замене условия эти поля обновляются, и `loadSavedCode()` загружает их в редактор.
- **Проверка принадлежности задачи к уроку** — `task.lesson_id != lesson_id` → 404,防止 доступ к задачам других уроков.
- **Редирект вместо 404 для закрытых уроков** — при прямом переходе на закрытый/истёкший урок выполняется редирект на `/lessons`, а не ошибка (более дружелюбный UX).

## 6. Исправленные ошибки

В ходе работы над подзадачей 7 обнаружены и исправлены:

| Проблема | Решение |
|---|---|
| `changeLanguage()` вызывалась в `editor.html`, но не была определена → JS-ошибка при смене языка | Функция добавлена в `lesson.html`, переключает режим CodeMirror |
| Подсветка синтаксиса отсутствовала (`vendor/` пуст) — нарушение раздела 6.4 ТЗ | CodeMirror 5.65.16 скачан и подключён (Python + C++ режимы) |
| `loadSavedCode()` вызывалась до инициализации CodeMirror | Порядок изменён: `initEditor()` → `loadSavedCode()` внутри `initEditor()` |
| `submitCode()` читал код из textarea, а не из CodeMirror | Добавлена проверка: `editor ? editor.getValue() : textarea.value` |
