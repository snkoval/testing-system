|---|---|---|

### 6. Submission (раздел 8.1.6)

| Атрибут | Тип | Ограничения |

## Модуль управления тестами (`app/test_files.py`)

Тесты хранятся в файлах, а не в БД (раздел 7.2 ТЗ).

### Структура файлов

```
tests_files/
└── <task_id>/
    ├── in_1.txt      # входные данные теста 1
    ├── out_1.txt     # ожидаемый результат теста 1
    ├── in_2.txt      # входные данные теста 2
    ├── out_2.txt     # ожидаемый результат теста 2
    └── ...
```

### Функции

| Функция | Описание |
|---|---|
| `get_test_dir(task_id)` | Возвращает `Path` к директории тестов задачи; создаёт директорию при отсутствии |
| `add_test(task_id, number, input, output)` | Создаёт файлы `in_N.txt` и `out_N.txt` |
| `update_test(task_id, number, input, output)` | Перезаписывает существующие файлы теста |
| `delete_test(task_id, number)` | Удаляет файлы теста (молча, если не существует) |
| `get_tests(task_id)` | Возвращает список `(number, input, output)`, отсортированный по номеру |
| `get_test_count(task_id)` | Возвращает количество тестов задачи |
| `get_examples(task_id)` | Возвращает первые 2 теста `[(input_1, output_1), (input_2, output_2)]` для автогенерации примеров в условии |

## Каскадные связи

```
Group ──┬── Student ──── Submission
        │                  ↑
        │              Task
        │                ↑
        └── Lesson ──── Task ── Submission
```

- Удаление **Group** → удаляются все **Student** → удаляются все **Submission**
- Удаление **Lesson** → удаляются все **Task** → удаляются все **Submission**
