# Система тестирования программ учеников

Веб-приложение для автоматической проверки решений задач на Python и C++. Учитель создаёт уроки и задачи с тестами, ученики пишут код в браузере (CodeMirror) и отправляют на проверку. Результаты доступны в таблице с экспортом в Excel.

**Стек:** Python (Flask) + SQLite (SQLAlchemy) + HTML/CSS/JavaScript.

---

## Возможности

- **Два типа пользователей** — учитель (администратор) и ученик.
- **Управление группами и учениками** — создание групп, массовое добавление учеников с автоматической генерацией логинов и паролей.
- **Управление уроками** — порядок, теория, задачи (A, B, C…), управление доступом по времени.
- **Задачи с тестами** — описание, форматы ввода/вывода, лимиты времени и памяти, набор тестов.
- **Песочница** — безопасное выполнение кода Python и C++ с ограничениями по времени и памяти.
- **Таблица результатов** — прогресс всех учеников по всем задачам, экспорт в Excel (.xlsx).
- **CSRF-защита** — все POST-запросы защищены токеном.

## Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/snkovaleng/testing-system.git
cd testing-system
```

### 2. Виртуальное окружение

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка конфигурации

```bash
cp instance/config.example.py instance/config.py
```

Отредактируйте `instance/config.py` — укажите:

- **`SECRET_KEY`** — случайная строка (сгенерировать: `python -c "import secrets; print(secrets.token_hex(32))"`).
- **`TEACHER_USERNAME`** и **`TEACHER_PASSWORD`** — учётные данные учителя.
- **`SQLALCHEMY_DATABASE_URI`** — путь к базе данных (по умолчанию `instance/testing.db`).

> Настройки из `instance/config.py` перекрывают значения по умолчанию из `app/config.py`.

### 5. Инициализация базы данных

```bash
python init_db.py
```

Скрипт создаст базу данных, директории `uploads/` и `tests_files/`, а также учётную запись учителя.

### 6. Запуск

**Режим разработки:**

```bash
python run.py
```

Приложение будет доступно по адресу http://127.0.0.1:5000.

**Режим production (Windows — waitress):**

```bash
python wsgi.py
```

Приложение будет доступно по адресу http://0.0.0.0:8000.

**Режим production (Linux — gunicorn):**

```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:8000 wsgi:app
```

## Руководство пользователя (для учителя)

### Первый вход

1. Откройте в браузере адрес приложения (например, http://127.0.0.1:5000).
2. Войдите как учитель, используя логин и пароль из `instance/config.py` (по умолчанию `admin` / `admin123`).

### Создание групп и учеников

1. **Группы** → «Создать группу» — введите название (например, `ИП-21`).
2. Откройте группу → «Массовое добавление» — введите ФИО учеников (по одному в строке). Система автоматически сгенерирует логины и пароли.
3. Нажмите на ученика, чтобы увидеть или сбросить его пароль.

### Создание уроков и задач

1. **Уроки** → «Создать урок» — укажите порядковый номер, название, ссылку на теорию.
2. Откройте урок → «Задачи» → «Создать задачу» — заполните описание, форматы ввода/вывод, лимиты.
3. Откройте задачу → «Тесты» — загрузите тесты (файлы `.in` / `.out` или ввод в форме).

### Управление доступом

1. Откройте урок → «Доступ».
2. Откройте урок на нужное количество дней или бессрочно. Ученики увидят урок и смогут отправлять решения.

### Просмотр результатов

1. **Результаты** — таблица: строки — ученики, столбцы — задачи. В ячейках — количество пройденных тестов.
2. «Экспорт в Excel» — скачать таблицу в формате `.xlsx`.

## Тестирование

```bash
python -m pytest
```

Тесты используют in-memory SQLite. Для C++-тестов требуется установленный `g++` (без него эти тесты пропускаются).

---

## Безопасность

- Пароли учителя хранятся в виде хэша (Werkzeug).
- Пароли учеников хранятся в открытом виде (для отображения учителю) + хэш для проверки входа.
- CSRF-токен проверяется на всех POST/PUT/DELETE запросах.
- Код учеников выполняется в изолированной песочнице с ограничениями по времени и памяти.
