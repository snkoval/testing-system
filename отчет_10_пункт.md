## 1. Что реализовано

### 1.1. Страница результатов — `GET /teacher/results` (`app/teacher.py:557`)

| Параметр | Назначение |
|---|---|
| `group` (query, int) | ID выбранной группы |

**Алгоритм:**

1. Загрузка всех групп для выпадающего списка (`Group.query.order_by(Group.name)`).
2. Если `group` указан — построение матрицы результатов через `_build_results_matrix()`.
3. Группа не найдена → 404.
4. Рендеринг `teacher/results.html`.

### 1.2. Функция `_build_results_matrix(group_id)` (`app/teacher.py:506`)

Формирует матрицу «ученик × задача»:

| Шаг | Детали |
|---|---|
| Загрузка группы | `Group` не найдена → `None` |
| Ученики | `Student.query.filter_by(group_id=group_id).order_by(Student.seq_number)` |
| Задачи | Все уроки (`Lesson.query.order_by(Lesson.order_number)`), для каждого — задачи (`Task.query.order_by(Task.letter_index)`) |
| Отправки | `Submission.query.filter_by(student_id=s.id).all()` — словарь `{task_id: sub}` для каждого ученика |
| Число тестов | `get_test_count(t.id)` для каждой задачи |
| Заполнение ячеек | См. таблицу ниже |

**Логика ячеек:**

| Ситуация | Значение ячейки |
|---|---|
| Нет отправки | `''` (пусто) |
| Все тесты пройдены (`status == 'ok'`) | `'OK'` |
| Частичное / неполное решение | `'{passed}/{total}'` (например `1/2`) |
| Ошибка JSON-парсинга | `''` (пусто) |

**Структура результата:**

```python
{
    'group': Group,
    'students': [Student, ...],
    'tasks': [Task, ...],
    'matrix': [
        {'student': Student, 'cells': ['', 'OK', '1/2', ...]},
        ...
    ]
}
```

### 1.3. Заголовки столбцов (раздел 10.1 ТЗ)

Формат: `<тема урока> — <буква>. <название задачи>`

Пример: `Урок 1 — A. Сумма`

### 1.4. Экспорт в Excel — `GET /teacher/results/export` (`app/teacher.py:573`)

| Параметр | Назначение |
|---|---|
| `group` (query, int, обязательный) | ID группы |

**Валидация:**
- `group` отсутствует → **400**
- Группа не найдена → **404**

**Формирование `.xlsx` (через `openpyxl`):**

1. Создание книги, лист с названием группы.
2. Заголовки (строка 1): «Фамилия», «Имя», «Логин» + заголовки задач.
   - Шрифт: **жирный**, выравнивание по центру.
3. Данные (строки 2+): фамилия, имя, логин, значения ячеек.
4. Авторазмер колонок: ширина = `min(max_len + 2, 40)`.
5. Сохранение в `BytesIO`, возврат как attachment.

**Имя файла:** `<название_группы>_results.xlsx`

**MIME-тип:** `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`

## 4. Технические решения

- **Единая функция `_build_results_matrix()`** — используется и для HTML-страницы, и для Excel-экспорта. Это гарантирует идентичность данных в обоих форматах и упрощает поддержку.

- **`get_test_count()` для числа тестов** — если `test_results` JSON не содержит `total_tests`, используется реальное число тестов из файлового хранилища. Это обеспечивает корректность даже при изменении тестов после отправки.

- **`BytesIO` вместо временного файла** — Excel-файл генерируется в памяти и отправляется напрямую через `send_file()`. Не требует очистки временных файлов.

- **Авторазмер колонок с лимитом 40** — ширина колонки вычисляется по максимальной длине значения, но ограничена 40 символами во избежание слишком широких колонок.

- **Все уроки (открытые и закрытые)** — результаты включают задачи из всех уроков, так как таблица результатов предназначена для итогового анализа. Соответствует плану подзадачи 10.
