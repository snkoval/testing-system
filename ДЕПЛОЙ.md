# Развертывание на сервере (Ubuntu 24.04, Beget VPS)

## Исходные данные

- **Сервер:** 155.212.209.148, Ubuntu 24.04, 2 vCPU / 4 ГБ RAM / 38 ГБ NVMe
- **Приложение:** Flask + SQLite + песочница (Python, C++)
- **Репозиторий:** https://github.com/snkoval/testing-system.git

Все команды ниже выполняются по SSH под пользователем `root`, если не указано иное.

```bash
ssh root@155.212.209.148
```

## Этап 1. Подготовка системы

```bash

```

## Этап 5. Конфигурация приложения

### 5.1. Генерация SECRET_KEY

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

**Скопируйте результат** — он понадобится на следующем шаге.

### 5.2. Создание config.py

```bash
sudo -u testsys nano /opt/testing-system/instance/config.py
```

Вставьте (замените значения в `<...>`):

```python
import os

# Секретный ключ (вставьте сгенерированный).
SECRET_KEY = '<ВСТАВЬТЕ_СГЕНЕРИРОВАННЫЙ_КЛЮЧ_СЮДА>'

# База данных SQLite.
_SQLITE_DIR = os.path.dirname(os.path.abspath(__file__))
SQLALCHEMY_DATABASE_URI = 'sqlite:///' + os.path.join(_SQLITE_DIR, 'testing.db')

# Лимиты по умолчанию.
DEFAULT_TIME_LIMIT = 1
DEFAULT_MEMORY_LIMIT = 256

# Пути к интерпретаторам.
PYTHON_EXEC = 'python3'
CPP_COMPILER = 'g++'

# Учётные данные учителя — ОБЯЗАТЕЛЬНО измените!
TEACHER_USERNAME = 'admin'
TEACHER_PASSWORD = '<ПРИДУМАЙТЕ_НАДЕЖНЫЙ_ПАРОЛЬ>'

# CSRF-защита.
CSRF_PROTECTION_ENABLED = True
```

Сохраните (`Ctrl+O`, `Enter`) и закройте (`Ctrl+X`).

### 5.3. Инициализация базы данных

```bash
sudo -u testsys bash -c 'cd /opt/testing-system && source venv/bin/activate && python init_db.py'
```

> Должно появиться: `Учитель "admin" создан.` и `База данных успешно инициализирована.`

### 5.4. Проверка запуска

```bash
sudo -u testsys bash -c 'cd /opt/testing-system && source venv/bin/activate && gunicorn -w 3 -b 127.0.0.1:8000 wsgi:app --timeout 30'
```

Откройте **вторую SSH-сессию** и проверьте:

```bash
curl -I http://127.0.0.1:8000
```

> Должен вернуться `HTTP/1.1 200 OK`.

Остановите Gunicorn (`Ctrl+C`) в первой сессии. Переходим к настройке автозапуска.

## Этап 6. Настройка автозапуска (systemd)

Создайте файл службы:

```bash
nano /etc/systemd/system/testingsys.service
```

Вставьте:

```ini
[Unit]
Description=Testing System (Flask/Gunicorn)
After=network.target

[Service]
User=testsys
Group=testsys
WorkingDirectory=/opt/testing-system
Environment="PATH=/opt/testing-system/venv/bin"
ExecStart=/opt/testing-system/venv/bin/gunicorn -w 3 -b 127.0.0.1:8000 wsgi:app --timeout 30
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

Запуск и включение автозапуска:

```bash
systemctl daemon-reload
systemctl start testingsys
systemctl enable testingsys
systemctl status testingsys
```

> Статус должен быть `active (running)`. Если `failed` — смотрите логи:
> ```bash
> journalctl -u testingsys -n 50 --no-pager
> ```

Проверка:

```bash
curl -I http://127.0.0.1:8000
```

## Этап 7. Настройка Nginx

### 7.1. Конфигурация

```bash
nano /etc/nginx/sites-available/testingsys
```

Вставьте (пока без домена — по IP):

```nginx
server {
    listen 80;
    server_name 155.212.209.148;

    # Статические файлы отдаёт Nginx напрямую.
    location /static/ {
        alias /opt/testing-system/app/static/;
        expires 30d;
    }

    # Всё остальное — проксируется в Gunicorn.
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_redirect off;
    }

    client_max_body_size 10M;
}
```

### 7.2. Активация

```bash
ln -sf /etc/nginx/sites-available/testingsys /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl reload nginx
systemctl restart nginx
```

> `nginx -t` должен сообщить `syntax is ok` и `test is successful`.

### 7.3. Проверка

Откройте в браузере: **http://155.212.209.148**

Должна открыться страница приложения. Войдите как учитель (`admin` / ваш пароль из `config.py`).

## Этап 8. Подключение домена и SSL (опционально)

### 8.1. Регистрация домена

Купите домен `.ru` (у Beget или любого регистратора: Reg.ru, Ru-Center).

### 8.2. Настройка DNS

В панели регистратора добавьте **A-запись**:

| Тип | Имя | Значение |
|---|---|---|
| A | `@` | `155.212.209.148` |
| A | `www` | `155.212.209.148` |

> DNS обновляется от нескольких минут до 24 часов. Проверить:
> ```bash
> dig +short your-domain.ru
> ```
> Должен вернуть `155.212.209.148`.

### 8.3. Обновление конфигурации Nginx

```bash
nano /etc/nginx/sites-available/testingsys
```

Замените строку `server_name`:

```nginx
    server_name your-domain.ru www.your-domain.ru 155.212.209.148;
```

Перезагрузите:

```bash
systemctl reload nginx
```

### 8.4. Установка SSL-сертификата (Let's Encrypt — бесплатно)

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d your-domain.ru -d www.your-domain.ru
```

- На вопрос про email — укажите реальный (уведомления об истечении).
- Согласитесь с условиями (`Y`).
- Редирект с HTTP на HTTPS — выберите **Yes** (`2`).

> Certbot автоматически изменит конфиг Nginx и настроит автопродление.
> Сертификат действителен 90 дней, продлевается автоматически.

Проверьте автопродление:

```bash
certbot renew --dry-run
```

Откройте в браузере: **https://your-domain.ru**

## Безопасность

- Приложение работает от пользователя `testsys`, а не от `root`.
- Песочница выполняет код учеников с ограничениями по времени и памяти.
- Рекомендуется отключить SSH-вход по паролю и использовать ключи:
  ```bash
  nano /etc/ssh/sshd_config
  # Установить: PasswordAuthentication no
  systemctl restart sshd
  ```
  > Сначала убедитесь, что SSH-ключ настроен и работает — иначе потеряете доступ.
