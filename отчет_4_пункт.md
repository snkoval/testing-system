## 1. Что реализовано

### 1.1. Модель: `password_plain` в `Student`
Добавлено поле `password_plain` в модель `Student` (`app/models.py`) для хранения открытого пароля. Согласно ТЗ (раздел 3.2.4), учителю нужен доступ к паролям учеников в открытом виде для выдачи учётных данных.

| Поле | Тип | Описание |
|---|---|---|
| `password_plain` | `String(20)`, NOT NULL | Открытый пароль (макс. 6 символов) |

### 1.2. Утилиты (`app/utils.py`)
Создан модуль с двумя функциями:

| Функция | Описание |
|---|---|
| `generate_password(length=6)` | Генерация случайного пароля: строчные буквы (`a–z`), цифры (`0–9`), спецсимволы (`!@#$%^&*`). Без заглавных букв. Длина — 6 символов. |
| `generate_login(group_name, seq_number)` | Формирование логина по шаблону `<маска_группы>_<порядковый_номер>`. Например `7A_1gr_1`. |

### 1.3. Маршруты управления группами (`app/teacher.py`)

| Маршрут | Метод | Описание |
|---|---|---|
| `/teacher/groups` | GET | Список всех групп: название, количество учеников, действия (открыть, удалить) |
| `/teacher/groups/create` | GET, POST | Создание группы: форма с полями (номер класса, литера, номер группы), валидация маски `<класс>_<группа>gr`, проверка уникальности |
| `/teacher/groups/<id>` | GET | Просмотр группы: таблица учеников (логин, пароль в открытом виде, ФИО), формы добавления и управления паролями |
| `/teacher/groups/<id>/add` | POST | Добавление одного ученика: генерация логина и пароля, автоинкремент `seq_number` |
| `/teacher/groups/<id>/bulk-add` | GET, POST | Массовое создание: ввод списка `Фамилия Имя` по строкам, генерация логинов и паролей, отображение результата |
| `/teacher/groups/<id>/delete` | POST | Удаление группы с каскадным удалением учеников |
| `/teacher/students/<id>/password` | POST | Смена пароля ученика вручную |
| `/teacher/students/<id>/reset-password` | POST | Сброс пароля: автогенерация нового случайного пароля |

**Логика создания группы:**
- Номер класса: 1–2 цифры (regex `^\d{1,2}$`)
- Литера класса: одна заглавная буква, автоматически переводится в верхний регистр (regex `^[А-ЯЁA-Z]$`)
- Номер группы: одна цифра (regex `^\d$`)
- Итоговое название: `<класс><литера>_<группа>gr` (например `7A_1gr`, `10B_2gr`)
- Проверка уникальности перед записью в БД

**Логика добавления ученика:**
- `seq_number` вычисляется как `max(seq_number) + 1` по группе
- Логин: `generate_login(group.name, seq_number)`
- Пароль: `generate_password()`
- Хранятся оба: `password_hash` (для проверки) и `password_plain` (для отображения учителю)

## 4. Технические решения

- **`password_plain` хранится в БД** — это требование ТЗ: учителю нужна возможность видеть и выдавать пароли учеников в открытом виде (раздел 3.2.4). Вместе хранятся и `password_hash` (для auth), и `password_plain` (для отображения).
- **Валидация маски группы** через regex: `^\d{1,2}$` для класса, `^[А-ЯЁA-Z]$` для литеры, `^\d$` для номера группы. Строчная литера автоматически переводится в верхний регистр (удобство ввода).
- **`seq_number`** вычисляется через `db.func.max()` — корректно работает как для пустой группы, так и при добавлении к существующим ученикам.
- **Массовое создание** парсит textarea по строкам, пропускает некорректные (меньше 2 слов), продолжает нумерацию с существующих.
- **Каскадное удаление** работает через `cascade='all, delete-orphan'` в модели `Group` (реализовано в подзадаче 2).
