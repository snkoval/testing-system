## 1. Что реализовано

### 1.1. Функция `is_lesson_accessible` (`app/utils.py`)
Добавлена функция проверки доступности урока для учеников:

| Условие | Результат |
|---|---|
| `is_open = False` | Недоступен |
| `access_days = None` | Доступен бессрочно |
| `opened_at + access_days >= now` | Доступен |
| `opened_at + access_days < now` | Истёк, недоступен |

Обрабатывается случай, когда SQLite возвращает `opened_at` как naive-datetime — добавляется `tzinfo=utc` перед сравнением.

### 1.2. Маршруты управления уроками (`app/teacher.py`)

| Маршрут | Метод | Описание |
|---|---|---|
| `/teacher/lessons` | GET | Список всех уроков: порядковый номер, тема, статус (открыт/закрыт/истёк), срок доступа, действия |
| `/teacher/lessons/create` | GET, POST | Создание урока: порядковый номер, тема, ссылка на теорию (необязательно). Создаётся в статусе «закрыт» |
| `/teacher/lessons/<id>/edit` | GET, POST | Редактирование: изменение номера, темы, ссылки |
| `/teacher/lessons/<id>/access` | POST | Управление доступом: открыть (с указанием срока в днях или бессрочно) / закрыть |
| `/teacher/lessons/<id>/delete` | POST | Удаление урока с каскадным удалением задач, тестов, отправок |

**Логика управления доступом:**
- **Открыть:** `is_open=True`, `opened_at=now()`, `access_days` = число дней или `None` (бессрочно, если поле пустое)
- **Закрыть:** `is_open=False`

### 1.3. Список уроков для ученика (`app/student.py`)
Маршрут `/lessons` теперь:
- Загружает все уроки, отсортированные по убыванию порядкового номера (новые сверху — согласно ТЗ 5.2)
- Фильтрует через `is_lesson_accessible()` — ученик видит только открытые и неистёкшие уроки
- Каждый урок — ссылка на страницу урока

### 1.4. Шаблоны
Созданы/обновлены 3 шаблона:

| Шаблон | Назначение |
|---|---|
| `teacher/lessons.html` | Таблица уроков с действиями (редактировать, открыть/закрыть, удалить). Статус определяется через `is_lesson_accessible()` прямо в шаблоне |
| `teacher/lesson_form.html` | Форма создания/редактирования (один шаблон для обоих режимов, переключается по `lesson`) |
| `student/lessons.html` | Список доступных уроков в виде ссылок (новые сверху) |

### 1.5. Контекстный процессор (`app/__init__.py`)
Добавлен `@app.context_processor`, который делает `is_lesson_accessible` доступной во всех шаблонах (используется в `teacher/lessons.html` для отображения статуса «Открыт» / «Истёк»).

## 4. Технические решения

- **Naive datetime из SQLite:** SQLite теряет timezone при чтении `DateTime`-колонок. `is_lesson_accessible` проверяет `tzinfo` и добавляет UTC при необходимости.
- **Context processor:** `is_lesson_accessible` доступна глобально в шаблонах — статус «Открыт/Истёк» вычисляется прямо в `lessons.html`.
- **Сортировка для учеников:** `order_by(Lesson.order_number.desc())` — новые уроки сверху (ТЗ 5.2).
- **Пустое `access_days` = бессрочно:** Если поле оставлено пустым при открытии, `access_days = None` → урок доступен до явного закрытия.
- **Единая форма** `lesson_form.html` для создания и редактирования — определяется наличием `lesson` в контексте.
