## 1. Что реализовано

### 1.1. Эндпоинт отправки — `POST /submit` (`app/student.py:69`)

Эндпоинт связывает интерфейс ученика (Фрейм 3) с песочницей (`checker.py`) и БД (`Submission`).

| Параметр формы | Назначение |
|---|---|
| `task_id` | ID задачи (обязательный) |
| `code` | Исходный код ученика (обязательный) |
| `language` | `'python'` или `'cpp'` |
| `csrf_token` | CSRF-токен (Flask-WTF / SeaSurf) |

**Алгоритм работы:**

1. **Получение `task_id`** — если отсутствует → 404.
2. **Загрузка задачи** — `Task` не найден → 404.
3. **Проверка доступа к уроку** — загрузка `Lesson`, проверка `is_lesson_accessible()`. Если урок закрыт или истёк → **403**.
4. **Валидация кода** — пустой `code` → **400** (`jsonify({'error': '...'})`).
5. **Вызов песочницы** — `check_submission(code, language, task_id)`.
6. **Сохранение / обновление `Submission`**:
   - Если у ученика уже есть отправка для этой задачи — **перезаписывается** (`code`, `language`, `test_results`).
   - Если нет — создаётся новая запись.
   - Раздел 8.1.6 ТЗ: «код — только последняя версия».
7. **Возврат JSON** — результат `check_submission` отправляется клиенту.

### 1.2. Защита доступа

| Проверка | Код ответа |
|---|---|
| Эндпоинт требует входа ученика (`@login_required_student`) | 302 → `/login` |
| `task_id` отсутствует | 404 |
| Задача не существует | 404 |
| Урок закрыт или истёк | 403 |
| Пустой код | 400 |

### 1.3. Сохранение результата (раздел 8.1.6)

Модель `Submission` хранит **только последнюю попытку** для пары (ученик, задача):

```python
submission = Submission.query.filter_by(
    student_id=student_id, task_id=task_id
).first()

if submission:
    submission.code = code          # перезапись
    submission.language = language
    submission.test_results = json.dumps(result)
else:
    submission = Submission(...)    # новая запись
    db.session.add(submission)
```

`test_results` — JSON-строка с полной структурой результата песочницы (статус, число тестов, результаты по каждому тесту, ошибка компиляции).

### 1.4. Загрузка последней попытки (раздел 7.1, шаг 5 подзадачи 9)

При открытии задачи (AJAX `GET /lessons/<lesson_id>/tasks/<task_id>`) шаблон `task_statement.html` передаёт сохранённый код и язык через скрытые поля:

```html
{% if submission %}
<input type="hidden" id="saved-code" value="{{ submission.code }}">
<input type="hidden" id="saved-language" value="{{ submission.language }}">
{% endif %}
```

JavaScript-функция `loadSavedCode()` в `lesson.html`:
- Читает значения скрытых полей.
- Загружает код в редактор CodeMirror (`editor.setValue(code)`).
- Устанавливает язык в переключателе (`<select>`).
- Вызывает `changeLanguage()` для обновления подсветки.

### 1.5. Отображение результатов (раздел 7.4)

JavaScript-функция `submitCode()` в `lesson.html`:

1. Собирает код из CodeMirror и язык из `<select>`.
2. Отправляет `FormData` через `fetch('/submit', {method: 'POST'})`.
3. Рендерит ответ:

| Элемент | Реализация |
|---|---|
| Общее число тестов и пройденных | `data.passed + '/' + data.total_tests` |
| Итоговый статус | `data.status === 'ok'` → «Все тесты пройдены!», иначе «Есть ошибки» |
| Ошибки компиляции | `<pre>data.compile_error</pre>` |
| Статус по каждому тесту | Таблица: номер теста, иконка (OK/X), сообщение |

## 4. Технические решения

- **Перезапись вместо истории** — согласно разделу 8.1.6 ТЗ, хранится только последняя версия кода. Уникальное ограничение `(student_id, task_id)` обеспечивает одну запись на пару. Повторная отправка обновляет существующую запись через `query.filter_by().first()`.

- **403 для закрытого урока, а не 404** — задача существует, но доступ запрещён. Это даёт ученику понять, что код принят, но проверка отклонена по причине доступа.

- **400 для пустого кода через JSON** — `jsonify({'error': 'Код не может быть пустым'}), 400`. Это позволяет JavaScript обработать ошибку, а не получать HTML-страницу.

- **CSRF-токен в FormData** — `formData.append('csrf_token', csrf_token())`. Требуется глобальным CSRF-защитником (подзадача 11).

- **Сохранённый код через скрытые поля** — код передаётся в `task_statement.html` как `<input type="hidden">`. При AJAX-замене условия эти поля обновляются, и `loadSavedCode()` загружает их в редактор без дополнительного запроса.
