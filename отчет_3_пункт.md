## 1. Что реализовано

### 1.1. Модуль декораторов доступа (`app/decorators.py`)
Создан новый модуль с двумя декораторами:

| Декоратор | Назначение |
|---|---|
| `@login_required_teacher` | Проверяет `session['teacher_id']`, иначе редирект на `/admin` |
| `@login_required_student` | Проверяет `session['student_id']`, иначе редирект на `/login` |

### 1.2. Blueprint `auth` (`app/auth.py`)
Реализованы маршруты:

| Маршрут | Метод | Описание |
|---|---|---|
| `/admin` | GET, POST | Вход учителя: форма логин+пароль, проверка через `check_password_hash`, при успехе запись `session['teacher_id']`, редирект на `/teacher/` |
| `/login` | GET, POST | Вход ученика: форма логин+пароль, проверка через `check_password_hash`, при успехе запись `session['student_id']`, редирект на `/lessons` |
| `/logout` | GET | Выход: очищает сессию, редирект на `/admin` (если вышел учитель) или `/login` (если ученик) |

### 1.3. Blueprint `teacher` (`app/teacher.py`)
Реализованы маршруты:

| Маршрут | Метод | Защита | Описание |
|---|---|---|---|
| `/teacher/` | GET | `@login_required_teacher` | Панель учителя (дашборд с навигацией) |
| `/teacher/groups` | GET | `@login_required_teacher` | Заглушка списка групп (для будущей подзадачи 4) |
| `/teacher/settings` | GET, POST | `@login_required_teacher` | Смена пароля учителя |

**Смена пароля — логика:**
- Проверка старого пароля через `check_password_hash`.
- Проверка совпадения нового пароля и подтверждения.
- При ошибке — `flash` сообщение и повторный рендер формы.
- При успехе — обновление `password_hash`, редирект на `/teacher/settings`.

### 1.4. Blueprint `student` (`app/student.py`)
Реализован маршрут:

| Маршрут | Метод | Защита | Описание |
|---|---|---|---|
| `/lessons` | GET | `@login_required_student` | Заглушка списка уроков (для будущей подзадачи 5) |

### 1.5. Шаблоны
Созданы 7 HTML-шаблонов:

| Шаблон | Назначение |
|---|---|
| `auth/teacher_login.html` | Форма входа учителя |
| `auth/student_login.html` | Форма входа ученика |
| `teacher/dashboard.html` | Панель учителя |
| `teacher/groups.html` | Заглушка списка групп |
| `teacher/settings.html` | Форма смены пароля |
| `student/lessons.html` | Заглушка списка уроков |

Все шаблоны наследуются от `base.html`, используют `get_flashed_messages()` для отображения ошибок и успеха.

## 4. Технические решения

- **Пароли хранятся в виде хэша** через `werkzeug.security.generate_password_hash` / `check_password_hash`.
- **Сессии Flask** (`session`) используются для хранения авторизации — без дополнительной зависимости.
- **`db.session.get()`** вместо устаревшего `Query.get()` (SQLAlchemy 2.0).
- **Декораторы** вынесены в отдельный модуль `app/decorators.py` для переиспользования в подзадачах 4–5.
- **`/logout`** единый маршрут для обоих типов пользователей — определяет тип по содержимому сессии перед очисткой.
