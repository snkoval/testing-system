## 1. Техническое задание

### 1.1. Создание документа

Создан файл `Техническое_задание.md` на основе описания заказчика. Документ включает 13 разделов:

| № | Раздел | Содержание |
|---|---|---|
| 1 | Назначение и цели | Описание задач системы |
| 2 | Общие требования | Архитектура, роли, стек: Python (Flask), SQLite, HTML/CSS/JS |
| 3 | Режим учителя | Вход `/admin`, группы, ученики, маски логинов/паролей |
| 4 | Доступ ученика | Прямая ссылка, конфиденциальность данных |
| 5 | Уроки | Создание, список, срок доступа (дни / бессрочно) |
| 6 | Страница урока | 3 фрейма: список задач, условие (6 разделов), редактор кода |
| 7 | Тестирование программ | Процесс проверки, файлы тестов `in_N.txt`/`out_N.txt`, лимиты |
| 8 | Требования к данным | 6 сущностей: Учитель, Группа, Ученик, Урок, Задача, Результат |
| 9 | Интерфейс | UI для учителя и ученика |
| 10 | Таблица результатов | Сводная таблица по группам, экспорт в Excel |
| 11 | Безопасность | Хэши паролей, CSRF/XSS, песочница |
| 12 | Развёртывание | Независимые экземпляры на разных серверах |
| 13 | Уточнения и решения | Языки (Python, C++), только последняя версия кода, лимиты, стек |

### 1.2. Уточнения, внесённые заказчиком

- **Языки программирования:** Python (обновляемая версия) и C++ (GNU GCC Compiler)
- **История отправок:** сохраняется только последняя версия кода
- **Лимиты ресурсов:** настраиваются для каждой задачи (по умолчанию — 1 с / 256 КБ)
- **Срок доступа к урокам:** задаётся учителем (в днях или бессрочно)
- **Уведомления:** не требуются; формируется сводная таблица с экспортом в Excel
- **Стек:** Backend — Python (Flask), Frontend — HTML/CSS/JavaScript, СУБД — SQLite

### 1.3. Дополнительные правки заказчика

- Вход учителя: логин/пароль задаются при первичной настройке сервера
- Массовое создание учеников в группе
- Смена и сброс пароля ученика через учителя
- Подсветка синтаксиса в редакторе кода
- Редактирование тестов после публикации урока
- Тесты хранятся в файлах `in_<номер>.txt` / `out_<номер>.txt`
- Структура условия задачи — 6 разделов (лимиты, текст, ввод, вывод, примеры, пояснения)

### 1.4. Проверка на непротиворечивость

Проведена ревизия всех разделов. Найдены и исправлены 5 несоответствий:

| № | Проблема | Исправление |
|---|---|---|
| 1 | Маска группы `*??_?gr` нестандартна | Заменена на описание `<класс>_<группа>gr` |
| 2 | Нет минимума тестов (нужны для примеров) | Добавлен минимум — 2 теста |
| 3 | Дублирующая строка в разделе 6.3 | Удалена |
| 4 | Компилятор C++ указан непоследовательно | `GNU GCC Compiler` указан во всех разделах |
| 5 | Модель данных не отражает структуру условия | Добавлены поля: текст условия, ввод, вывод, пояснения |

## 4. Реализация подзадачи 1 — подготовка проекта

### 4.1. Созданные файлы и директории

```
testing-system/
├── app/
│   ├── __init__.py          # фабрика приложения create_app()
│   ├── config.py            # Config + TestConfig
│   ├── models.py            # модель Teacher (заглушка)
│   ├── auth.py              # blueprint auth
│   ├── teacher.py           # blueprint teacher (/teacher)
│   ├── student.py           # blueprint student
│   ├── checker.py           # blueprint checker (заглушка)
│   ├── templates/
│   │   └── base.html        # базовый шаблон
│   ├── static/
│   │   ├── css/style.css    # стили (синий/белый фон, контраст)
│   │   └── js/main.js       # пустой JS
│   └── uploads/.gitkeep     # директория для временных файлов
├── tests/
│   ├── __init__.py
│   ├── conftest.py          # фикстуры app, client
│   └── test_app.py          # 6 тестов
├── requirements.txt         # flask, flask-sqlalchemy, openpyxl, pytest
├── run.py                   # точка входа
├── init_db.py               # скрипт инициализации БД + учителя
└── .gitignore               # исключения (venv, __pycache__, instance, *.db)
```

### 4.2. Конфигурация (`app/config.py`)

- `SECRET_KEY`, `SQLALCHEMY_DATABASE_URI` (SQLite)
- Лимиты по умолчанию: `DEFAULT_TIME_LIMIT = 1`, `DEFAULT_MEMORY_LIMIT = 256`
- Пути к компиляторам: `PYTHON_EXEC`, `CPP_COMPILER`
- Правила генерации паролей: `PASSWORD_CHARS`, `PASSWORD_LENGTH = 6`
- Учётные данные учителя: `TEACHER_USERNAME`, `TEACHER_PASSWORD`
- `TestConfig` — in-memory SQLite, отключённый CSRF

### 4.3. Фабрика приложения (`app/__init__.py`)

- Функция `create_app(config_class)`:
  - Инициализация SQLAlchemy
  - Регистрация 3 blueprints: auth, teacher, student
  - Главная страница `/` → `base.html`

### 4.4. TDD-тесты (`tests/test_app.py`)

| Тест | Что проверяет |
|---|---|
| `test_app_creates_successfully` | Приложение создаётся без ошибок |
| `test_app_has_sqlalchemy_initialized` | SQLAlchemy инициализирован |
| `test_index_page_returns_200` | Главная страница возвращает HTTP 200 |
| `test_config_has_required_settings` | SECRET_KEY и URI БД заданы |
| `test_default_limits_configured` | Лимиты 1 с / 256 КБ |
